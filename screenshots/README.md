# Screenshot Placeholders

Screenshot folder for the phishing analysis workflow project.

---

## Project Purpose

This folder will hold sanitized screenshots that support the phishing workflow project.

Screenshots should show realistic triage evidence, completed documentation, or safe training examples. They should not include real user data, private tenant details, credentials, tokens, or message identifiers.

---

## Tools Used

- Microsoft 365 phishing review context
- Email header viewer or sanitized header scenario
- URL reputation review tool
- IOC checklist
- Incident notes template
- Screenshot/redaction tool as needed

---

## What I Practiced

- Planning screenshots that support a junior analyst workflow.
- Separating missing evidence from completed documentation.
- Defining safe redaction rules before public posting.
- Keeping the project honest about what evidence has and has not been added.

---

## Skills Demonstrated

- Phishing investigation documentation.
- Evidence organization.
- Screenshot planning.
- Data redaction awareness.
- Portfolio documentation discipline.

---

## Lessons Learned

Screenshots should support the investigation story. If the image does not help explain the workflow, it should not be included.

For a public portfolio, missing screenshots are better than unsafe or misleading screenshots.

## Screenshot Safety Rules

- Redact real names.
- Redact real email addresses.
- Redact tenant IDs.
- Redact message IDs.
- Redact private URLs.
- Redact tokens, cookies, and credentials.
- Use sanitized training data where possible.
