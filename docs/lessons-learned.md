# Lessons Learned

## Technical Lessons

- A suspicious email should be reviewed through multiple indicators, not one signal.
- Header authentication results help support the finding but do not replace full analysis.
- Link destination matters more than visible link text.
- User action determines containment urgency.
- Credential entry changes the response from message triage to possible account compromise.

## Operational Lessons

- Preserve evidence before taking action.
- Write notes that another analyst can understand quickly.
- Separate confirmed facts from assumptions.
- Escalate when user interaction, multiple recipients, or attachments increase risk.
- Keep documentation concise and repeatable.

## Interview-Defensible Takeaway

This project shows a practical junior SOC workflow: intake, evidence preservation, header review, IOC extraction, safe reputation checks, risk classification, containment recommendation, and incident documentation.
