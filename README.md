# Phishing Analysis Workflow

Portfolio project for junior SOC, security operations, and IT support to security transition roles.

Status: completed portfolio documentation; screenshot evidence polish pending.

This project documents a practical phishing investigation workflow using a Microsoft 365 context. It stays within junior analyst triage, documentation, and escalation practice.

---

## Project Purpose

The purpose of this completed portfolio project is to show how a junior analyst or IT support professional can receive a user-reported suspicious email, preserve evidence, review headers and links, extract indicators, classify risk, recommend containment, and write clear incident notes.

The project is built from sanitized phishing scenarios and is designed to be realistic, readable, screenshot-friendly, and interview-defensible.

## Lessons Learned

Phishing analysis is stronger when the review process is structured.

The useful part of the workflow is not only identifying a suspicious indicator. It is documenting the full decision path: what was checked, what evidence mattered, how risk was classified, and what should happen next.
